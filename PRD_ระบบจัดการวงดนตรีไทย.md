# Product Requirements Document (PRD)
## ระบบจัดการวงดนตรีไทย (Thai Music Ensemble Management System)

**เวอร์ชันเอกสาร:** 2.0
**วันที่อัปเดต:** 15 กรกฎาคม 2569
**เวอร์ชันก่อนหน้า:** 1.0 (30 มิถุนายน 2569)

---

## 1. ภาพรวมโปรเจกต์ (Overview)

ระบบเว็บแอปพลิเคชันสำหรับบริหารจัดการวงดนตรีไทย (ชมรม/ชุมนุมดนตรีไทยในสถาบันการศึกษา) ครอบคลุมการจัดการสมาชิก เพลงและโน้ตเพลง การจัดงานแสดง การจัดผังเวที ตารางซ้อม และคลังอุปกรณ์ เพื่อทดแทนกระบวนการที่เดิมทำผ่าน Excel และการสื่อสารแบบกระจัดกระจาย ให้รวมศูนย์ข้อมูลและลดความผิดพลาด เช่น การจองเครื่องดนตรีซ้ำซ้อนในเพลงเดียวกัน

### เป้าหมายหลัก
- ลดความซ้ำซ้อนและความผิดพลาดในการจัดสรรเครื่องดนตรีต่อเพลง
- ให้สมาชิกเห็นภาพรวมงานแสดง วันซ้อม และสถานะของตนเองได้ในที่เดียว
- ให้ Admin ควบคุมและตรวจสอบย้อนหลังการเปลี่ยนแปลงผังเวทีและตารางซ้อมได้
- แปลงข้อมูลโน้ตเพลงจาก Excel เดิมเข้าสู่ระบบดิจิทัลโดยไม่เปลี่ยนรูปแบบที่ผู้ใช้คุ้นเคย

---

## 2. กลุ่มผู้ใช้งาน (User Roles)

| Role | คำอธิบาย |
|---|---|
| สมาชิก (Member) | ผู้เล่นดนตรีทั่วไป ดูเพลง, ตอบ RSVP, จองคิวเครื่องดนตรี, ดูตารางซ้อม |
| หัวหน้าเครื่อง (Head) | สิทธิ์เหมือนสมาชิก + แก้ไขผังเวที/จองคิวได้เฉพาะเครื่องดนตรีของตน |
| Admin | สิทธิ์เต็มทุกส่วน รวมจัดการสมาชิก ตารางซ้อม audit log |

---

## 3. ขอบเขตฟังก์ชัน (Functional Requirements)

### 3.1 ระบบสมาชิก (Authentication & Profile)

- FR-1.1: ผู้ใช้สมัครสมาชิกใหม่และเข้าสู่ระบบ (Login)
- FR-1.2: โปรไฟล์สมาชิก: ชื่อเล่น, รุ่น, เครื่องดนตรีหลัก, เครื่องดนตรีรอง
- FR-1.3: ประวัติการแสดงย้อนหลังของสมาชิก — บันทึกอัตโนมัติจากระบบจองคิวเครื่องดนตรี

---

### 3.2 หน้า Dashboard

- FR-2.1: สรุปงานแสดงที่กำลังจะถึง
- FR-2.2: สรุปวันซ้อมที่ใกล้ถึง — Calendar interactive แสดง performance date และ practice day รวมกัน

---

### 3.3 หน้าเพลง (Song Library)

- FR-3.1: ค้นหาเพลงแบบ non-case sensitive
- FR-3.2: กรองเพลงตามประเภท (genre/category)
- FR-3.3: ข้อมูลต่อเพลง: ชื่อเพลง, ประเภท, โน้ตเพลง, เวลาที่ใช้เล่น
- FR-3.4: Micro-adjust — Admin แก้ไขโน้ตเพลงทีละช่องได้โดยตรงจากหน้าเว็บ
- FR-3.5: Song Version History — บันทึกทุกการแก้ไขโน้ตเพลง พร้อม restore เวอร์ชันก่อนหน้าได้

#### 3.3.1 ระบบ Import เพลง

- FR-3.6: Import จาก Excel แบบ bulk พร้อม preview และแจ้งเตือนแถวผิดรูปแบบ
- FR-3.7: กรอกโน้ตเพลงด้วยมือเป็นกริด: 1 บรรทัด = 8 ห้อง, 1 ห้อง = 4 ช่อง
- FR-3.8: แต่ละช่องรับ text อิสระ รองรับอักขระและสัญลักษณ์ไทย (ด ร ม ฟ ซ, ตัวเลข, `-`, สัญลักษณ์หายใจ)

---

### 3.4 หน้าการแสดง (Performance)

#### 3.4.1 การจัดการงานแสดง

- FR-4.1: ปฏิทินกิจกรรมแสดงงานแสดงทั้งหมด
- FR-4.2: สร้างงานแสดงใหม่ เลือกได้หลายวันต่องาน พร้อมระบุนักแสดงที่เข้าร่วม
- FR-4.3: ค้นหาและเพิ่มเพลงเข้างานแสดง (non-case sensitive, กรองตามประเภท)
- FR-4.4: แสดงรายการเพลงทั้งหมดที่ถูกเพิ่มเข้างาน
- FR-4.5: RSVP — สมาชิกยืนยันสถานะว่าง/ไม่ว่าง แยกเป็นรายวันได้
- FR-4.6: Admin เพิ่ม/ลบสมาชิกเข้า/ออกรายการงานแสดงโดยตรง
- FR-4.7: Performance Head Assignment — กำหนดหัวหน้าแยกต่อ performance

#### 3.4.2 การจัดลำดับเพลง (Running Order)

- FR-4.8: เลือกสมาชิกที่เล่นในแต่ละเพลงและเลือกเครื่องดนตรีให้แต่ละคน

#### 3.4.3 ระบบจองคิวเครื่องดนตรีต่อเพลง

- FR-4.9: สมาชิกเลือกเครื่องดนตรีที่จะเล่นในแต่ละเพลงของงานแสดง
- FR-4.10: Validation ป้องกันการจองซ้ำซ้อน (ยกเว้นเครื่องที่กำหนดให้เล่นพร้อมกันได้หลายคน)

#### 3.4.4 ผังเวที (Stage Layout)

- FR-4.11: สร้างผังเวทีแบบ dynamic กำหนดขนาดเองได้ (เช่น 10x5)
- FR-4.12: ลากเครื่องดนตรีวางบนผังเวที ขนาดไอคอนปรับสัดส่วนตามขนาดเวที
- FR-4.13: ค้นหาตำแหน่งเครื่องดนตรีใน Stage Editor ได้
- FR-4.14: Version History ทุกครั้งที่แก้ไขผัง พร้อมย้อนดูและกู้คืนเวอร์ชันก่อนหน้า
- FR-4.15: บันทึกผังเวทีเป็น Template เพื่อนำกลับมาใช้ซ้ำกับงานแสดงอื่น

---

### 3.5 สิทธิ์ Admin

- FR-5.1: Admin ดูข้อมูลสมาชิกทั้งหมด (ชื่อเล่น, รุ่น, เครื่องดนตรีหลัก-รอง)
- FR-5.2: สร้างตารางซ้อม เลือกงานแสดงที่จะซ้อมและกำหนดวันที่ซ้อม
- FR-5.3: Role-based Access Control — Head แก้ไขได้เฉพาะของตน, Admin แก้ได้ทุกส่วน
- FR-5.4: Audit Log บันทึกผู้แก้ไข วันเวลา เชื่อมกับ Version History ผังเวที (append-only)

---

### 3.6 ตารางซ้อม (Practice Schedule)

- FR-6.1: ตารางซ้อมแบบละเอียด แบ่งเป็น PracticeDay / PracticeSlot / PracticeMemberGroup รองรับกลุ่มย่อยภายในการซ้อม
- FR-6.2: Availability Grid — สมาชิกระบุความพร้อมรายช่วงเวลาในแต่ละวันซ้อม
- FR-6.3: Collapsible sections — ซ่อน/แสดงกลุ่มซ้อมได้

---

### 3.8 สมุดโน้ตเพลง (Sheet Music Editor)

ระบบแก้ไขโน้ตเพลงในรูปแบบตาราง หน้าตาและการใช้งานคล้าย Excel แต่ **ไม่มีการคำนวณสูตร** — ใช้สำหรับจดและแสดงโน้ตเพลงไทยเป็นหลัก

#### โครงสร้างตาราง

- FR-8.1: ตารางโน้ตมีค่าเริ่มต้น 8 คอลัมน์ต่อแถว
- FR-8.2: รองรับการพิมพ์ภาษาไทย ภาษาอังกฤษ และอักขระพิเศษ เช่น สระ/วรรณยุกต์ซ้อน (ฺ ํ ็ ่ ้ ๊ ๋)

#### ฟังก์ชันการแก้ไข

- FR-8.3: Merge Cell — รวมเซลล์ได้ทั้งแนวนอนและแนวตั้ง
- FR-8.4: Highlight — ไฮไลต์สีพื้นหลังเซลล์ได้
- FR-8.5: ปรับขนาดคอลัมน์และแถวได้อิสระ
- FR-8.6: ปรับขนาดตัวอักษรต่อเซลล์ได้
- FR-8.7: เปลี่ยนฟอนต์ได้ (ค่าเริ่มต้น: Sarabun รองรับภาษาไทย)
- FR-8.8: จัดตำแหน่งข้อความ — ซ้าย / กลาง / ขวา
- FR-8.9: ตัวหนา (Bold) / ตัวเอียง (Italic) / ขีดเส้นใต้ (Underline)
- FR-8.10: เพิ่ม/ลบแถวและคอลัมน์ได้

#### การส่งออก

- FR-8.11: Export โน้ตเพลงเป็นไฟล์ `.xlsx` ได้ พร้อมรักษาการจัดรูปแบบ (สี, merge, ขนาดคอลัมน์)

---

### 3.7 รายการอุปกรณ์ (Equipment Inventory)

- FR-7.1: ตารางคลังอุปกรณ์: ID, รายการ, จำนวน, ชนิด, สภาพการใช้งาน, ขนาด (ยาว/กว้าง/สูง ซม.), หมายเหตุ
- FR-7.2: Global Instrument Equipment — ตารางอุปกรณ์ที่ผูกกับเครื่องดนตรีแต่ละชนิดในระดับ global

---

## 4. ความต้องการที่ไม่ใช่ฟังก์ชัน (Non-Functional Requirements)

- NFR-1: ระบบต้องรองรับการพิมพ์ภาษาไทยในทุกฟิลด์ รวมถึงฟิลด์โน้ตเพลงที่มีอักขระและสัญลักษณ์พิเศษ
- NFR-2: ระบบ Version History และ Audit Log ต้องไม่สามารถลบหรือแก้ไขย้อนหลังได้ (append-only)
- NFR-3: ระบบต้องรองรับการใช้งานพร้อมกันหลายคนในระบบจองคิวเครื่องดนตรี โดย validation ต้องทำงานแบบ real-time หรือใกล้เคียง เพื่อป้องกันการจองซ้ำซ้อน

---

## 5. ความเสี่ยงและข้อควรพิจารณา (Open Issues)

| หัวข้อ | รายละเอียด | สถานะ |
|---|---|---|
| โครงสร้างคอลัมน์อุปกรณ์ | ความหมายของ "รายการ (B-C รวมกัน)" ต้องอ้างอิงไฟล์ Excel ต้นฉบับ | รอข้อมูล |
| เครื่องดนตรีที่เล่นพร้อมกันได้ | ต้องกำหนด list ของเครื่องดนตรีที่ยกเว้นจาก validation การจองซ้ำ (เช่น ระนาดทุ้ม) | รอยืนยันรายชื่อทั้งหมด |
