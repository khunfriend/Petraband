# Product Requirements Document (PRD)
## ระบบจัดการวงดนตรีไทย (Thai Music Ensemble Management System)

**เวอร์ชันเอกสาร:** 3.0
**วันที่อัปเดต:** 22 กรกฎาคม 2569
**เวอร์ชันก่อนหน้า:** 2.0 (15 กรกฎาคม 2569)

---

## 1. ภาพรวมโปรเจกต์ (Overview)

ระบบเว็บแอปพลิเคชันสำหรับบริหารจัดการวงดนตรีไทย (ชมรม/ชุมนุมดนตรีไทยในสถาบันการศึกษา) ครอบคลุมการจัดการสมาชิก เพลงและโน้ตเพลง การจัดงานแสดง การจัดผังเวที ตารางซ้อม และคลังอุปกรณ์ เพื่อทดแทนกระบวนการที่เดิมทำผ่าน Excel และการสื่อสารแบบกระจัดกระจาย ให้รวมศูนย์ข้อมูลและลดความผิดพลาด เช่น การจองเครื่องดนตรีซ้ำซ้อนในเพลงเดียวกัน

### เป้าหมายหลัก
- ลดความซ้ำซ้อนและความผิดพลาดในการจัดสรรเครื่องดนตรีต่อเพลง
- ให้สมาชิกเห็นภาพรวมงานแสดง วันซ้อม และสถานะของตนเองได้ในที่เดียว
- ให้ Admin ควบคุมและตรวจสอบย้อนหลังการเปลี่ยนแปลงผังเวทีและตารางซ้อมได้
- แปลงข้อมูลโน้ตเพลงจาก Excel เดิมเข้าสู่ระบบดิจิทัลโดยไม่เปลี่ยนรูปแบบที่ผู้ใช้คุ้นเคย

### เทคโนโลยีที่ใช้ (Tech Stack)

| ชั้น | เทคโนโลยี |
|---|---|
| Framework | Next.js 16 (App Router) + React 19 |
| ภาษา | TypeScript |
| ฐานข้อมูล | PostgreSQL 15+ ผ่าน Prisma 7 ORM |
| Authentication | NextAuth 5 (Credentials + bcryptjs) |
| Styling | Tailwind CSS 4 |
| Excel I/O | `xlsx` |
| Word Export | `docxtemplater` + `pizzip` |
| Image capture | `html-to-image` (สำหรับ export ผังเวที) |
| Email | Resend (สำหรับ password reset) |

## 2. กลุ่มผู้ใช้งาน (User Roles)

| Role | คำอธิบาย |
|---|---|
| สมาชิก (MEMBER) | ผู้เล่นดนตรีทั่วไป ดูเพลง, ตอบ RSVP, จองคิวเครื่องดนตรี, ดูตารางซ้อม |
| หัวหน้าเครื่อง (HEAD) | สิทธิ์เหมือนสมาชิก + แก้ไขผังเวที/จัดคิวได้เฉพาะงานแสดงที่ตนถูกกำหนดเป็นหัวหน้า |
| Admin (ADMIN) | สิทธิ์เต็มทุกส่วน รวมจัดการสมาชิก ตารางซ้อม audit log |

สถานะบัญชี: `ACTIVE` / `EXPIRED` — รองรับ **บัญชีชั่วคราว (Temporary Account)** สำหรับผู้ร่วมแสดงเฉพาะกิจ พร้อม utility สำหรับล้างบัญชีที่หมดอายุ

## 3. ขอบเขตฟังก์ชัน (Functional Requirements)

### 3.1 ระบบสมาชิก (Authentication & Profile)

- FR-1.1: ผู้ใช้สมัครสมาชิกใหม่และเข้าสู่ระบบ (Login) ผ่าน NextAuth Credentials
- FR-1.2: โปรไฟล์สมาชิก: ชื่อเล่น, รุ่น, เครื่องดนตรีหลัก, เครื่องดนตรีรอง, รูปโปรไฟล์ (avatar upload)
- FR-1.3: ประวัติการแสดงย้อนหลังของสมาชิก — บันทึกอัตโนมัติจากระบบจองคิวเครื่องดนตรี
- FR-1.4: Password Reset ผ่านอีเมล (ใช้ Resend ในการส่ง)
- FR-1.5: บัญชีชั่วคราว (Temporary Account) สำหรับผู้ร่วมแสดงที่ไม่ได้เป็นสมาชิกถาวร

---

### 3.2 หน้า Dashboard

- FR-2.1: สรุปงานแสดงที่กำลังจะถึง
- FR-2.2: Calendar interactive แสดง performance date และ practice day รวมกัน (พร้อม localization ปี พ.ศ.)

---

### 3.3 หน้าเพลง (Song Library)

- FR-3.1: ค้นหาเพลงแบบ non-case sensitive (ค้นตามชื่อหรือรหัสเพลง)
- FR-3.2: กรองเพลงตามประเภท (genre/category)
- FR-3.3: ข้อมูลต่อเพลง: ชื่อเพลง, ประเภท, โน้ตเพลง, เวลาที่ใช้เล่น, **BPM เริ่มต้น (defaultBpm)**, **time signature เริ่มต้น (defaultTimeSig)**
- FR-3.4: Micro-adjust — Admin แก้ไขโน้ตเพลงทีละช่องได้โดยตรงจากหน้าเว็บ (`PATCH /api/songs/[id]/sheet`)
- FR-3.5: Song Version History — บันทึกทุกการแก้ไขโน้ตเพลง พร้อม restore เวอร์ชันก่อนหน้าได้

#### 3.3.1 ระบบ Import เพลง

- FR-3.6: Import จาก Excel (`.xlsx`) แบบ bulk ผ่าน drag-and-drop พร้อม preview และแจ้งเตือนแถวผิดรูปแบบ
- FR-3.7: กรอกโน้ตเพลงด้วยมือเป็นกริด: 1 บรรทัด = 8 ห้อง, 1 ห้อง = 4 ช่อง
- FR-3.8: แต่ละช่องรับ text อิสระ รองรับอักขระและสัญลักษณ์ไทย (ด ร ม ฟ ซ, ตัวเลข, `-`, สัญลักษณ์หายใจ)

---

### 3.4 หน้าการแสดง (Performance)

#### 3.4.1 การจัดการงานแสดง

- FR-4.1: ปฏิทินกิจกรรมแสดงงานแสดงทั้งหมด
- FR-4.2: สร้างงานแสดงใหม่ เลือกได้หลายวันต่องาน (PerformanceDate) พร้อมระบุนักแสดงที่เข้าร่วม
- FR-4.3: ค้นหาและเพิ่มเพลงเข้างานแสดง (non-case sensitive, กรองตามประเภท)
- FR-4.4: แสดงรายการเพลงทั้งหมดที่ถูกเพิ่มเข้างาน
- FR-4.5: RSVP — สมาชิกยืนยันสถานะว่าง/ไม่ว่าง **แยกเป็นรายวัน (per PerformanceDate)**
- FR-4.6: Admin เพิ่ม/ลบสมาชิกเข้า/ออกรายการงานแสดงโดยตรง (PerformanceMember)
- FR-4.7: Performance Head Assignment — กำหนดหัวหน้าแยกต่อ performance (PerformanceHead)
- FR-4.8: **Export รายงานงานแสดงเป็นไฟล์ Word (.docx)** พร้อมฝังภาพผังเวที (html-to-image)

#### 3.4.2 การจัดลำดับเพลง (Running Order)

- FR-4.9: กำหนดลำดับเพลงในงานแสดง (PerformanceSong.order)
- FR-4.10: เลือกสมาชิกที่เล่นในแต่ละเพลงและเลือกเครื่องดนตรีให้แต่ละคน (SongAssignment)

#### 3.4.3 ระบบจองคิวเครื่องดนตรีต่อเพลง

- FR-4.11: สมาชิกเลือกเครื่องดนตรีที่จะเล่นในแต่ละเพลงของงานแสดง
- FR-4.12: Validation ป้องกันการจองซ้ำซ้อน (ยกเว้นเครื่องที่กำหนดให้เล่นพร้อมกันได้หลายคน) — *สถานะ: ยังไม่ implement, ดูหัวข้อ 5*

#### 3.4.4 ผังเวที (Stage Layout)

- FR-4.13: สร้างผังเวทีแบบ dynamic กำหนดขนาดเองได้ (widthUnits × heightUnits) พร้อมกำหนดหน่วย (unitLabel)
- FR-4.14: ลากเครื่องดนตรีวางบนผังเวที ขนาดไอคอนปรับสัดส่วนตามขนาดเวที
- FR-4.15: กำหนดชื่อเครื่องดนตรีและ footprint แบบ custom ได้
- FR-4.16: Equipment setup ต่อ stage item (ผูกอุปกรณ์เข้ากับตำแหน่งบนเวที)
- FR-4.17: กำหนดสมาชิกไปยัง stage item แต่ละตำแหน่งได้
- FR-4.18: Version History ทุกครั้งที่แก้ไขผัง (StageLayoutVersion) พร้อมย้อนดูและกู้คืนเวอร์ชันก่อนหน้า
- FR-4.19: บันทึกผังเวทีเป็น Template (StageTemplate) เพื่อนำกลับมาใช้ซ้ำกับงานแสดงอื่น

---

### 3.5 สิทธิ์ Admin

- FR-5.1: Admin ดูข้อมูลสมาชิกทั้งหมด (ชื่อเล่น, รุ่น, เครื่องดนตรีหลัก-รอง) พร้อม filter/search
- FR-5.2: สร้างตารางซ้อม เลือกงานแสดงที่จะซ้อมและกำหนดวันที่ซ้อม
- FR-5.3: Role-based Access Control — HEAD แก้ไขได้เฉพาะงานที่ตนถูก assign, ADMIN แก้ได้ทุกส่วน
- FR-5.4: Password Reset โดย Admin
- FR-5.5: Audit Log บันทึกผู้แก้ไข วันเวลา เชื่อมกับ Version History (append-only) — *สถานะ: schema พร้อม แต่ยังไม่ wire เข้า endpoint, ดูหัวข้อ 5*

---

### 3.6 ตารางซ้อม (Practice Schedule)

- FR-6.1: ตารางซ้อมแบบละเอียด แบ่งเป็น PracticeSchedule → PracticeDay → PracticeSlot → PracticeMemberGroup รองรับกลุ่มย่อยภายในการซ้อม
- FR-6.2: Availability Grid — สมาชิกระบุความพร้อมรายช่วงเวลาในแต่ละวันซ้อม (PracticeAvailability)
- FR-6.3: Collapsible sections — ซ่อน/แสดงกลุ่มซ้อมได้
- FR-6.4: รองรับหลาย schedule ต่อ 1 งานแสดง
- FR-6.5: Summary endpoint สำหรับตรวจสอบการตอบรับ (`/api/practice-schedules/[id]/summary`)

---

### 3.7 รายการอุปกรณ์ (Equipment Inventory)

- FR-7.1: ตารางคลังอุปกรณ์: ID, รายการ, จำนวน, จำนวนที่ชำรุด (brokenQuantity), ชนิด, ขนาด (ยาว/กว้าง/สูง ซม.), หมายเหตุ
- FR-7.2: Global Instrument Equipment — ตารางอุปกรณ์ที่ผูกกับเครื่องดนตรีแต่ละชนิดในระดับ global (ชื่อ / จำนวนเก้าอี้ / จำนวนโต๊ะ ต่อเครื่องดนตรี)

> **หมายเหตุ:** ระบบยืม-ให้ยืมอุปกรณ์ (Borrow/Lend, ยืม vs ให้ยืม) เคยพัฒนาไว้แต่ถูก **revert ออกจาก main แล้ว** (commit 61d1d5b) — ปัจจุบันไม่มี field เกี่ยวกับ loan ใน Equipment model

---

### 3.8 สมุดโน้ตเพลง (Sheet Music Editor)

ระบบแก้ไขโน้ตเพลงในรูปแบบตาราง หน้าตาและการใช้งานคล้าย Excel แต่ **ไม่มีการคำนวณสูตร** — ใช้สำหรับจดและแสดงโน้ตเพลงไทยเป็นหลัก

#### โครงสร้างตาราง

- FR-8.1: ตารางโน้ตมีค่าเริ่มต้น 8 คอลัมน์ต่อแถว (ปรับเปลี่ยนได้)
- FR-8.2: รองรับการพิมพ์ภาษาไทย ภาษาอังกฤษ และอักขระพิเศษ เช่น สระ/วรรณยุกต์ซ้อน (ฺ ํ ็ ่ ้ ๊ ๋)

#### ฟังก์ชันการแก้ไข

- FR-8.3: Merge Cell — รวมเซลล์ได้ทั้งแนวนอนและแนวตั้ง (MergedCell)
- FR-8.4: Highlight — ไฮไลต์สีพื้นหลังเซลล์ได้
- FR-8.5: ปรับขนาดคอลัมน์และแถวได้อิสระ (ColumnWidth / RowHeight)
- FR-8.6: ปรับขนาดตัวอักษรต่อเซลล์ได้
- FR-8.7: เปลี่ยนฟอนต์ได้ (ค่าเริ่มต้น: Sarabun รองรับภาษาไทย)
- FR-8.8: จัดตำแหน่งข้อความ — ซ้าย / กลาง / ขวา
- FR-8.9: ตัวหนา (Bold) / ตัวเอียง (Italic) / ขีดเส้นใต้ (Underline)
- FR-8.10: เพิ่ม/ลบแถวและคอลัมน์ได้

#### การส่งออก

- FR-8.11: Export โน้ตเพลงเป็นไฟล์ `.xlsx` ได้ พร้อมรักษาการจัดรูปแบบ (สี, merge, ขนาดคอลัมน์)

## 4. ความต้องการที่ไม่ใช่ฟังก์ชัน (Non-Functional Requirements)

- NFR-1: ระบบต้องรองรับการพิมพ์ภาษาไทยในทุกฟิลด์ รวมถึงฟิลด์โน้ตเพลงที่มีอักขระและสัญลักษณ์พิเศษ
- NFR-2: ระบบ Version History และ Audit Log ต้องไม่สามารถลบหรือแก้ไขย้อนหลังได้ (append-only)
- NFR-3: ระบบต้องรองรับการใช้งานพร้อมกันหลายคนในระบบจองคิวเครื่องดนตรี โดย validation ต้องทำงานแบบ real-time หรือใกล้เคียง เพื่อป้องกันการจองซ้ำซ้อน
- NFR-4: การแสดงวันที่ทั้งหมดใน UI ใช้ปฏิทินไทย (พ.ศ.)

## 5. ช่องว่างและงานที่ยังคั่งค้าง (Known Gaps & Open Issues)

| หัวข้อ | รายละเอียด | สถานะ |
|---|---|---|
| Anti double-booking (FR-4.12) | ยังไม่มี unique constraint / validation กันการจองเครื่องดนตรีซ้ำใน SongAssignment | **ยังไม่ implement** |
| Audit Log wiring (FR-5.5) | Prisma model `AuditLog` มีอยู่แล้ว แต่ยังไม่มี endpoint ใดเรียกเขียน log | **ยังไม่ implement** |

| ระบบยืม-ให้ยืมอุปกรณ์ | ถูก revert ออกจาก main (commit 61d1d5b) — รอทบทวนความต้องการก่อนพัฒนาใหม่ | pending decision |
